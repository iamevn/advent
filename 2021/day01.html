Day 1 of Advent of Code 2021.

Part 1

Given a series of numbers, part 1 asks for a count of how many numbers are greater than the previous number.

We're given an example and the expected result for this example: 

measurements : [Nat]
 
measurements : [Nat]
measurements = [199, 200, 208, 210, 200, 207, 240, 269, 260, 263]
increases : Nat
 
increases : Nat
increases = 7

We can write a function to do so recursively: 

countIncreases : [Nat] -> Nat
 
countIncreases : [Nat] -> Nat
countIncreases =
  use Nat <
  use List +:
  go : Nat -> [Nat] -> Nat
  go total = cases
    [a, b] ++ rest 
      | a < b     -> go (Nat.increment total) (b +: rest)
      | otherwise -> go total (b +: rest)
    _ -> total
  go 0

And check that it produces the expected result for the example: 

countIncreases measurements Nat.== increases
⧨
true

I wrote some quick helpers to read lines from a file:

  getLinesRec : [Text] ->{IO, Exception} [Text]
  getLinesRec acc =
    if isFileEOF h then acc else getLinesRec (acc :+ io.getLine h)

If you're familiar with Clojure,|>is similar to its->>macro. It lets us more clearly write composed functions without needing a bunch of parentheses. The results of {fileLines filename} are passed as an argument to {filterMap Nat.fromText} and the result of that is passed tocountIncreasesand so on.

We usefilterMapbecauseNat.fromTextreturns an {Optional Nat}.filterMapwill give us a {[Nat]} throwing out anyNonefrom applyingNat.fromText .

Part 2

Part 2 asks us to compare three-element sliding windows instead of single elements. 

199  A      
200  A B    
208  A B C  
210    B C D
200  E   C D
207  E F   D
240  E F G  
269    F G H
260      G H
263        H

Each three-element window is denoted by a letter here. Instead of comparing `199` and `200`, we instead compare window A (`199 + 200 + 208 = 607`) and window B (`200 + 208 + 210 = 618`) to find that the measurements increased from A to B.

We don't actually care about the sums in the windows and can just check the border elements. For that same example, windows A and B share `200` and `208`, we just check the first measurement in A (`199`) against the last element in B (`210`). Since `210` is greater than `199`, the sum of window B must be greater than the sum of window A. 

slidingWindowIncreases : [Nat] -> Nat
 
slidingWindowIncreases : [Nat] -> Nat
slidingWindowIncreases =
  go : Nat -> [Nat] -> Nat
  go total = cases
    a +: rest ->
      use Nat <
      match rest with
        [_, _, b] ++ _  
          | a < b      -> go (Nat.increment total) rest
          | otherwise  -> go total rest
        _              -> total
    _         -> total
  go 0

Slightly different thancountIncreases , compare the first and fourth elements to see if we should increase the total accumulator as we walk through the list. We return the total that's been accumulating when there's less than 4 elements to match against. 

example.slidingWindowIncreases : Nat
 
example.slidingWindowIncreases : Nat
example.slidingWindowIncreases = 5
slidingWindowIncreases measurements
  Nat.== example.slidingWindowIncreases
⧨
true
